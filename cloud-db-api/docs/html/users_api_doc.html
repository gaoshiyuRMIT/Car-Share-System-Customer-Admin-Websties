Users API Endpoints¶

POST /users/register¶

Register a new user.

Example request

POST /users/register HTTP/1.0
Content-Type: application/json

{
    "username": "shiyugao1",
    "password": "qi8H8R7OM4xMUNMPuRAZxlY.",
    "fName": "Shiyu",
    "lName": "Gao",
    "email": "shiyugao1@test.css.com"
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "success": true
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): usermame"
}
Request JSON Object

  • username – username

  • password – hashed password

  • fName – first name

  • lName – last name

  • email – email

Response JSON Object

  • data – has the key “success”, which has a boolean value, indicating whether the new user is successfully added

  • error_code – a short code name for the error

  • error_message – readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

POST /users/login¶

Check the log-in credentials.

Example request

POST /users/login HTTP/1.0
Content-Type: application/json

{
    "username": "shiyugao1",
    "password": "qi8H8R7OM4xMUNMPuRAZxlY."
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "success": true
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): passsword"
}
Request JSON Object

  • username – username

  • password – hashed password

Response JSON Object

  • data – has the key “success”, which is whether true or false, indicating whether the credential-check has passed

  • error_code – a short name for the error

  • error_message – a readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints
