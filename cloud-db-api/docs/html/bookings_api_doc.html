Bookings API Endpoints¶

GET /bookings/(int: booking_id)¶

Get the data of one booking.

Example request

GET /bookings/7 HTTP/1.0

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "booking_id": 7,
        "car_id": 1,
        "date_booking": "2019-01-01",
        "date_return": "2019-01-05",
        "status": "finished",
        "time_booking": "19:00:00",
        "time_return": "11:00:00",
        "username": "janedoe1"
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "MissingKey",
    "error_message": "The specified booking_id does not exist."
}
Response JSON Object

  • data – a json object representing a booking, with the keys “booking_id”, “username”, “car_id”, “date_booking”, “time_booking”, “date_return”, “time_return”, “status”. The entries are ordered first by status ( booked and inProgress first) and then by booking_id.

  • error_code (string) – a short code name for the error

  • error_message (string) – readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

POST /bookings/search¶

Search for bookings.

Example request

POST /bookings/search HTTP/1.0
Content-Type: application/json

{
    "username": "janedoe1",
    "status": "inProgress"
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": [
        {
            "booking_id": 2,
            "car_id": 1,
            "date_booking": "2020-05-04",
            "date_return": "2019-05-10",
            "status": "inProgress",
            "time_booking": "19:58:02",
            "time_return": "11:00:00",
            "username": "janedoe1"
        }
    ]
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): bodyy_type"
}
Request JSON Object

  • booking_id (int) –

  • username (string) –

  • car_id (int) –

  • date_booking (string) – in iso format, e.g. “2020-06-01”

  • time_booking (string) – in iso format, e.g. “19:58:02”

  • date_return (string) – in iso format, e.g. “2020-06-01”

  • time_return (string) – in iso format, e.g. “19:58:02”

  • status (string) – one of booked, inProgress, cancelled, finished

Response JSON Object

  • data – a list of booking datas, each booking with the keys “booking_id”, “username”, “car_id”, “date_booking”, “time_booking”, “date_return”, “time_return”, “status”. The entries are ordered first by status ( booked and inProgress first) and then by booking_id.

  • error_code (string) – a short code name for the error

  • error_message (string) – readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

PUT /bookings/(int: booking_id)/update¶

Update a booking’s data.

Example request

PUT /bookings/3/update HTTP/1.0
Content-Type: application/json

{
    "status": "cancelled"
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "success": true
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): statuss"
}
Request JSON Object

  • date_booking (string,optional) – in iso format, e.g. “2020-06-01”

  • time_booking (string,optional) – in iso format, e.g. “19:58:02”

  • date_return (string,optional) – in iso format, e.g. “2020-06-01”

  • time_return (string,optional) – in iso format, e.g. “19:58:02”

  • status (string,optional) –

Response JSON Object

  • data – has the key “success”, which is whether true or false

  • error_code – a short name for the error

  • error_message – a readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

POST /bookings/add¶

Add a new booking, and get the inserted booking’s booking_id.

Example request

POST /cars/add HTTP/1.0
Content-Type: application/json

{
    "car_id": 1,
    "date_booking": "2020-06-01",
    "date_return": "2020-06-05",
    "time_booking": "19:58:02",
    "time_return": "11:00:00",
    "username": "janedoe1"
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "booking_id": 4
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "MissingKey",
    "error_message": "The specified user does not exist"
}
Request JSON Object

  • username (string) – must be of an existing user

  • car_id (int) – must be of an existing car

  • date_booking (string) – in iso format, e.g. “2020-06-01”; must be in the future

  • time_booking (string) – in iso format, e.g. “19:58:02”; must be in the future

  • date_return (string) – in iso format, e.g. “2020-06-01”; must be in the future

  • time_return (string) – in iso format, e.g. “19:58:02”; must be in the future

Response JSON Object

  • data – has the key “booking_id”, which is the booking_id of the newly added booking

  • error_code – a short name for the error

  • error_message – a readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints
