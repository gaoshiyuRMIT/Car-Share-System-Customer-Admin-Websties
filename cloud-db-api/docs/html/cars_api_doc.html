Cars API Endpoints¶

GET /cars/(int: car_id)¶

Get the data of one car.

Example request

GET /cars/2 HTTP/1.0

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "body_type": "Sedan",
        "car_colour": "white",
        "car_id": 2,
        "car_model": "Audi S3",
        "cost_hour": 0.5,
        "latitude": -37,
        "longitude": 144,
        "num_seats": 4,
        "status": "available",
        "year": 2015
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "MissingKey",
    "error_message": "The specified car_id does not exist."
}
Response JSON Object

  • data – a json object representing a car, with the keys “car_id”, “year”, “car_model”, “body_type”, “num_seats”, “car_colour”, “cost_hour”, “latitude”, “longitude”, “status”.

  • error_code (string) – a short code name for the error

  • error_message (string) – readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

POST /cars/search¶

Search for cars.

Example request

POST /cars/search HTTP/1.0
Content-Type: application/json

{
    "car_colour": "white",
    "num_seats": [3, 5]
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": [
        {
            "body_type": "Sedan",
            "car_colour": "white",
            "car_id": 1,
            "car_model": "Audi S3",
            "cost_hour": 0.5,
            "latitude": -37,
            "longitude": 144,
            "num_seats": 4,
            "status": "available",
            "year": 2015
        }
    ]
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): bodyy_type"
}
Request JSON Object

  • car_id (int) – car ID

  • year – year of production. A list of length 2 (specifying range), or a single int (specifying exact value).

  • car_model (string) – car model

  • body_type (string) – body type

  • num_seats – number of seats. A list of length 2 (specifying range), or a single int (specifying exact value).

  • car_colour (string) – car colour

  • cost_hour – cost in AUD per hour. A list of length 2 (specifying range), or a single decimal number (specifying exact value).

Response JSON Object

  • data – a list of car datas, each json object respresenting a car, with the keys “car_id”, “year”, “car_model”, “body_type”, “num_seats”, “car_colour”, “cost_hour”, “latitude”, “longitude”, “status”.

  • error_code (string) – a short code name for the error

  • error_message (string) – readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

PUT /cars/(int: car_id)/update¶

Update a car’s data.

Example request

PUT /cars/12/update HTTP/1.0
Content-Type: application/json

{
    "car_colour": "silver grey"
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "success": true
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): carr_colour"
}
Request JSON Object

  • year (int,optional) – year of production

  • car_model (string,optional) – car model

  • body_type (string,optional) – body type

  • num_seats (int,optional) – number of seats

  • car_colour (string,optional) – colour

  • cost_hour (decimal,optional) – cost per hour

  • latitude (decimal,optional) – between -90 and 90

  • longitude (decimal,optional) – between -180 and 180

  • status (string,optional) – one of available, booked

Response JSON Object

  • data – has the key “success”, which is whether true or false

  • error_code – a short name for the error

  • error_message – a readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints

POST /cars/add¶

Add a new car, and get the inserted car’s car_id.

Example request

POST /cars/add HTTP/1.0
Content-Type: application/json

{
    "body_type": "Sedan",
    "car_colour": "white",
    "car_model": "Audi S3",
    "cost_hour": 0.5,
    "latitude": -37,
    "longitude": 144,
    "num_seats": 4,
    "status": "available",
    "year": 2015
}

Example response

HTTP/1.0 200 OK
Content-Type: application/json

{
    "data": {
        "car_id": 13
    }
}

Example erroneous response

HTTP/1.0 400 BAD REQUEST
Content-Type: application/json

{
    "error_code": "InvalidArgument",
    "error_message": "Invalid key(s): carr_colour"
}
Request JSON Object

  • year (int) – year of production

  • car_model (string) – car model

  • body_type (string) – body type

  • num_seats (int) – number of seats

  • car_colour (string) – colour

  • cost_hour (decimal) – cost per hour

  • latitude (decimal) – between -90 and 90

  • longitude (decimal) – between -180 and 180

  • status (string) – one of available, booked

Response JSON Object

  • data – has the key “car_id”, which is the car_id of the newly added car

  • error_code – a short name for the error

  • error_message – a readable error message

Status Codes

  • 200 OK – no error

  • 400 Bad Request – request data is wrong, either with wrong keys or the values do not conform to type/format constraints
